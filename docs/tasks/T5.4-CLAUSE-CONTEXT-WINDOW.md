## 做什么

### 唯一改动：补 `_infer_clause_type` 关键词表

`app/engine/clause_splitter.py` 第 166-172 行。**不新建文件、不改 kernel.py、不改 rules.json。**

当前是空壳：

```python
def _infer_clause_type(clause: ClauseBlock) -> tuple[str, float]:
    return _DEFAULT_TYPE, 0.0
```

改为：

```python
_CLAUSE_TYPE_KEYWORDS = [
    ("保修",   ["保修", "质保", "缺陷责任", "防水", "管线"]),
    ("付款",   ["付款", "支付", "结算", "进度款", "预付款", "工程款", "发票", "拨款"]),
    ("验收",   ["验收", "竣工", "交付", "移交", "接收", "组织验收"]),
    ("违约",   ["违约", "罚金", "罚款", "逾期", "赔偿", "责任"]),
    ("担保",   ["担保", "保证金", "保函", "履约", "投标"]),
    ("质保金", ["质保金", "质量保证金", "保修金"]),
    ("安全",   ["安全", "事故", "防护", "危险"]),
    ("争议",   ["争议", "仲裁", "诉讼", "管辖"]),
    ("工期",   ["工期", "开工", "竣工日期", "合同工期"]),
    ("工程概况", ["工程概况", "工程名称", "工程地点", "承包范围"]),
]

def _infer_clause_type(clause: ClauseBlock) -> tuple[str, float]:
    text = clause.clause_title + " " + clause.content[:200]
    for type_name, keywords in _CLAUSE_TYPE_KEYWORDS:
        if any(kw in text for kw in keywords):
            return type_name, 0.9
    return "其他", 0.0
```

**为什么不需要新文件和 kernel.py 改动**：

- `_infer_clause_type` 在 `_make_block()` 时被调用，直接把 `block.clause_type` 设成固定名
- `_apply_classifier()` 只在 `split()` 传入 classifier 参数时才覆盖——T5.4 不传 classifier，关键词表是唯一类型来源
- rules.json 的 clause_type 字段已就位，不需要再改
- kernel.py 调用 `ClauseSplitter.split(text)` 不传 classifier，_infer_clause_type 自动生效
