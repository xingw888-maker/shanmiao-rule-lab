## 第二部分 B：两个高收益修复

### B1：修复 cn-006 回归（幻觉，conf=0.85）

**问题确认**：文本"质量保证金应在缺陷责任期满后14天内返还。质量保证金担保函。"→ 模型看到"质量保证金"推理 3%（conf=0.85）。**提高全局阈值到 0.9 无效——0.85 仍然落在可接受区间的低端但不能一刀切，且 7 个 FIXED 全部 conf=0.95，调阈值不会误杀。**

**精确改法（不提升全局阈值，加固 prompt）**：

在 `road2_llm_extractor.py:56` cn-006 的 guidance 中追加：
```
"，文本中必须有百分号(%)或"百分之"字样才能提取"
```

同时在 `road2_llm_extractor.py:92` prompt 末尾的规则区域追加：
```
7. 提取百分比(%)、万分比(万分之)时, source_text 必须明确包含原始数值表达(百分号、千分号、万分之等)。禁止根据关键词推导默认值。
```

**验收**：`python tests/road2_compare.py` 全量，cn-006-FP-01 不再出现 REGRESSION。

### B2：修复 cn-020 field_rule_lookup 映射

**问题确认**：对照报告中 cn-020-POS-01 的 `n_structured_extractions=1` 但 `extract_value=null`。LLM 返回的字段名是"履约担保"，而 `road2_compare.py` 的 `extract_for_sample()` 方法中 `FIELD_LABELS_BY_RULE` 的键是"履约担保比例"——"履约担保" ≠ "履约担保比例"，字面匹配失败。

**改法**：修改 `tests/road2_compare.py:196` 附近的 fuzzy fallback 逻辑，在原有子串匹配基础上增加前缀匹配：

```python
# Current:
if r_lb in label or label in r_lb: rid = r_id; break

# Add before or after:
if r_lb.startswith(label) or label.startswith(r_lb): rid = r_id; break
```

或者在 `FIELD_LABELS_BY_RULE` 中增加 alias 字段 `"履约担保": "cn-020"`。

**注意**：这不会自动让 cn-020-POS-01 的 regex 路径也通过——regex 路径的 MISSING 可能是真的 context_pattern 不匹配。但至少 extracted 路径应该能走到 handler，让 handler 做数值比较。如果 extracted 路径仍然 MISSING，则问题在 handler 侧。

**验收**：`python tests/road2_compare.py` 全量，cn-020-POS-01 的 extracted_status 不再 MISSING（至少变成 PASSED/FAILED/NOT_APPLICABLE 之一）。
