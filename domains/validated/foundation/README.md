# Foundation — 跨域通用规则层

## 定位

Foundation 层是独立于行业领域的通用合同合规规则集合。它不依赖建工、购销或任何特定领域——这些规则适用于几乎所有商业合同类型。

## 分级逻辑

```
Foundation 层（通用） → 所有合同类型自动加载
   ↓
Domain 层（领域）    → 建工特有（竣工验收、安全生产、禁止转包等）
   ↓
Specific 层（特异）  → 特定合同类型的额外覆盖（未来可扩展）
```

三层规则叠加验证：一条合同文本跑 Foundation + 对应 Domain 的组合规则集。

## 规则来源

全部 14 条规则从 construction 域 v1.1.0 的 30 条规则中审计抽取。

抽取标准：
- 规则条件不依赖建工领域特有概念（如工程款、施工工艺、安全规范）
- 规则适用于购销合同、NDA 等非建工域
- 使用的 condition_type 为现有三种（numeric_comparison / required_pattern / forbidden_pattern）

## 五大主题

| 主题 | 规则数 | 覆盖内容 |
|------|--------|----------|
| 保修 (warranty) | 5 | 保修期限、质量保修书、缺陷责任期、质保期起算 |
| 争议解决 (dispute) | 3 | 诉讼/仲裁、管辖机构、协商前置 |
| 付款条件 (payment) | 3 | 付款节点条件、付款期限、逾期利息上限 |
| 违约金 (damages) | 2 | 计算基数、总额上限 |
| 质量标准 (quality) | 1 | 禁止低于强制性标准 |

## 与领域层的关系

- Foundation 层规则是**补充**而非替代——建工域 30 条规则中仍有 16 条领域特异性规则保留在原包中（竣工验收、安全生产、禁止转包、禁止垫资、履约担保、投标保证金）
- 加载一个领域时，Foundation + Domain 两套规则合并验证
- 新增 NDA 域时，直接复用 Foundation 层，只需写领域特异规则

## 维护原则

- Foundation 层只增加真正跨域通用的规则
- 新增规则前先审计：这个条件是否在购销/NDA/其他域合同中也合理？
- 领域特有规则留在各自 domain 包内，不向上抽取
